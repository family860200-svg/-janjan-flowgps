# 存檔 Skill（save-progress）v2.0 — Wave 架構

> 對話結束時說「存檔」「結束」「掰掰」即自動執行

---

## ⚡ Wave 架構總覽

**核心原則**：Wave N 完成才進 Wave N+1。緊急中斷 → 至少完成 Wave 1 + 收尾。

| Wave | 時間 | 內容 | 強制性 |
|------|------|------|--------|
| **Wave 1** | ~3 分鐘 | 偵察 + 核心產出 | 🔴 不可省略 |
| **Wave 2** | ~5 分鐘 | 人脈 + 洞察 + 草稿 | 🟠 盡量完成 |
| **收尾** | ~3 分鐘 | 待辦 + 快照 + Git | 🔴 不可省略 |

---

## 🌊 Wave 1（~3 分鐘）

### Step 0：確認時間 + 防重複檢查

```bash
date "+%Y-%m-%d %H:%M"
git -C "/Users/user/-janjan-flowgps" status --short
```

**防重複存檔**：

```bash
MARKER="/tmp/save-progress-done"
if [ -f "$MARKER" ]; then
  AGE=$(( $(date +%s) - $(stat -f "%m" "$MARKER") ))
  LAST=$(cat "$MARKER")
  if [ "$AGE" -lt 7200 ]; then
    echo "⚠️ 本 session 已存過（$LAST，${AGE}s 前）"
  fi
fi
```

- 已存過 → 跳過 Step 2（對話總結），直接 Step 3（散落待辦）+ 收尾
- JANJAN 明確說「重新存」→ 忽略 marker，全跑

### Step 1：讀取隨手一記

讀取 `F｜行動聚焦漏斗/manual_notes.json` 當天的記錄，納入對話摘要「✏️ 隨手一記」區塊。

### Step 2：生成對話總結

**存放**：`F｜行動聚焦漏斗/對話摘要/YYMMDD_主題.md`

| 對話時長 | 模板 |
|----------|------|
| < 10 分鐘 | 簡化版（5 區塊） |
| 10-60 分鐘 | 標準版（7 區塊） |
| > 60 分鐘 | 完整版（9 區塊） |

**標準版格式**：

```markdown
# YYMMDD_主題

> 時間：HH:MM－HH:MM（共 X 分鐘）

## ✅ 完成了什麼

## ❌ 未完成

## 📍 下一步行動

## ✏️ 隨手一記

## 💰 八大財富帳戶加分

| 帳戶 | 標籤 | 說明 |
|------|------|------|
| 成長 | 🌱 | |
| 技能 | ⚒️ | |
| ... | | |

## 📊 DRIP 分類

| 時段 | 活動 | DRIP | 財富 |
|------|------|------|------|
| HH:MM-HH:MM | [活動] | I | 🌱⚒️ |
```

### Step 3：掃描散落待辦（Loose-ends）

回溯本次對話，找出：

- 🔴 JANJAN 提出但沒執行的：「這個要改」「之後再處理」「先放著」
- 🔴 做到一半被打斷的任務
- 🟡 AI 建議但沒有後續回應的
- 🟡 提到但沒展開的：「對了 X 也要處理」
- 🟢 等外部回覆 / 條件觸發的

**輸出**：

- 有找到 → 列出後問：「以上 X 個 loose-end 要加進待辦清單嗎？（說『加』或『跳過』）」
- 沒找到 → 輸出 `✅ 無散落待辦`

---

## 🌊 Wave 2（~5 分鐘）

### Step 4：人脈觀察

掃描本次對話中提到的人名（家人、同事、客戶、社群朋友）：

- 有新互動 / 關係更新 → 更新 `L｜資源放大槓桿/人脈資料庫/人脈資料庫.md`
- 有承諾或等待事項 → 加入待辦清單
- 無新人脈訊號 → 輸出 `✅ 人脈：無新訊號`

---

### Step 5：AI 團隊學習記錄

掃描本次對話哪個 AI 角色有出力：

| 角色 | 負責範疇 | 記什麼 |
|------|----------|--------|
| 阿工 | 程式、自動化、系統 | 新建或修改的工具 |
| 阿文 | Threads、知識卡片、文案 | 發文素材、草稿 |
| 小書 | 手工書設計、市集 | 設計決定、進度 |
| 阿娟二號 | 配抄、客訴、業務 | 流程更新、計算結果 |

有角色出力 → 在 `L｜資源放大槓桿/AI團隊/[角色]/` 更新記錄（如有 LOG.md）。

---

### Step 6：Threads 草稿掃描

四鏡頭掃描（每類寫「有/無」）：

1. **反直覺洞察**：跟常識相反的結論？
2. **實戰紀錄**：有具體數字或前後對比？
3. **踩坑經驗**：做錯了什麼、怎麼修正？
4. **方法論**：可重複使用的框架？

**有素材 → 直接生成草稿**，存到 `F｜行動聚焦漏斗/對話摘要/YYMMDD_主題.md` 末尾（或獨立存）：

```markdown
## 📝 Threads 草稿

### 版本 A
[草稿]

### 版本 B（可選）
[草稿]
```

草稿生成後問：「⚡ 要趁熱發嗎？→ 說『發』直接走 /threads；說『存』先放著。」

## 🔴 收尾（~3 分鐘）

### Step 7：更新待辦清單

把「❌ 未完成」「📍 下一步」「散落待辦」全部寫入：

```
F｜行動聚焦漏斗/玩家待辦任務.md
```

**HARD GATE**：對話總結每一條「❌ 未完成」和「📍 下一步行動」都必須出現在待辦清單。逐項比對，缺一補一。

---

### Step 8：更新遊戲進度快照

**位置**：`F｜行動聚焦漏斗/game-progress.md`

更新：
1. `最後更新` 時間戳
2. 主線任務進度（有變化才改）
3. 能量狀態（今日完成的 Boss、身體、心情）
4. 近期重要日期（完成的打刪除線，新的補上去）

---

### Step 9：Git 備份

```bash
cd "/Users/user/-janjan-flowgps"
git add .
git commit -m "$(cat <<'EOF'
YYMMDD 時段存檔：[主題]

- [完成項目 1]
- [完成項目 2]

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
git push
```

**Commit 完成後必須回報**：

```
✅ 已推上 GitHub
📌 最新 commit：[hash] [message]
🕐 時間：HH:MM
```

Push 失敗必須說明原因，不可靜默跳過。

---

### Step 10：輸出存檔摘要 + 寫入防重複標記

```markdown
✅ 存檔完成 | HH:MM-HH:MM（X 分鐘）| [主題]
- 📄 總結：YYMMDD_主題.md
- 📍 待辦：已更新 X 項
- 💰 財富：[帳戶標籤]
- 🤝 人脈：[人名 / 無]
- 📝 草稿：[主題 / 無]
- 📦 commit：[hash]
```

然後寫入防重複標記：

```bash
date '+%H:%M' > /tmp/save-progress-done
```

## Error Handling

| 狀況 | 處理 |
|------|------|
| `nothing to commit` | 正常，跳過 commit，仍輸出摘要 |
| push 失敗 | `git pull --rebase` 後重試，失敗說明原因 |
| manual_notes.json 無今日記錄 | 「✏️ 隨手一記：無」|
| Wave 2 時間不夠 | 跳過 Step 4-6，列入下次補做 |

**最低限度**（緊急中斷時）：
1. 對話總結（Step 2）
2. 待辦更新（Step 7）
3. Git commit（Step 9）
