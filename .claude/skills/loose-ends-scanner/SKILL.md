# Loose Ends Scanner — 對話散落待辦掃描器

> 安裝方法：把這個檔案複製到 `.claude/skills/loose-ends-scanner/SKILL.md`

對話結束前或中途，掃描叉出去但沒收尾的行動項目，加進玩家行動中心，避免 10 件事裡漏 3 件。

---

## 觸發條件

- 說「有沒有漏掉什麼」「叉出去的」「散落待辦」「loose ends」
- save-progress 前主動執行（建議但非強制）
- 對話超過 1 小時且話題切換 3 次以上

---

## 掃描方法

### Step 1：回溯對話，找出所有行動訊號

逐段掃描對話內容，標記以下訊號：

| 訊號類型 | 範例 | 權重 |
|---------|------|------|
| **你提出但沒執行** | 「這個要改」「之後再處理」「先放著」 | 🔴 高 |
| **AI 宣稱完成但無實際記錄** | AI 說「✅ 已更新」但找不到對應操作 | 🔴 高 |
| **做到一半被打斷** | 開始做 A，話題切換到 B，A 沒收尾 | 🔴 高 |
| **AI 建議但你沒回應** | 「要不要我也...」然後話題切換 | 🟡 中 |
| **提到但沒展開** | 「對了 X 也要處理」然後繼續原話題 | 🟡 中 |
| **等外部回覆** | 「等 XX 回覆」「XX 確認後再...」 | 🟢 低（記錄即可） |
| **條件觸發** | 「如果 XX 就要做 YY」 | 🟢 低（記錄即可） |

### Step 2：比對已知系統，排除已處理的

排除清單（不重複加入）：
- 已在 `F｜🕹️｜玩家行動中心.md` 或 `F｜行動聚焦漏斗/玩家待辦任務.md` 的項目
- 已在本次對話總結「📍 下一步行動」的項目
- 已在 `game-progress.md` P0/P1/P2 的項目
- 本次對話中已標記 ✅ 完成的項目

### Step 3：分類 + 展示

```markdown
🔍 散落待辦掃描結果（共 X 項）

🔴 做到一半 / 你明確提到要做：
1. [描述] — 對話中哪段提到的
2. [描述] — 對話中哪段提到的

🟡 AI 建議但沒回應 / 提到但沒展開：
3. [描述] — 對話中哪段提到的

🟢 等回覆 / 條件觸發（記錄即可）：
4. [描述] — 觸發條件是什麼

已排除（已在行動中心或對話總結）：
- [項目] ← 已在 [位置]

要加入哪些到玩家行動中心？（全加 / 選擇 / 跳過）
```

### Step 4：寫入玩家行動中心

確認後，寫入 `F｜行動聚焦漏斗/玩家待辦任務.md`：

```markdown
- [ ] [描述] [首次：M/D] [來源：對話 #N loose-ends]
```

分類規則：
- 🔴 高權重 → P0 或 P1（看急迫程度）
- 🟡 中權重 → P1 或 P2
- 🟢 低權重 → Parking 或對應區塊

## 使用場景

### 場景 1：對話結束前快速掃
```
你：有沒有漏掉什麼？
Claude：🔍 掃描中... [輸出結果]
```

### 場景 2：搭配 save-progress 使用
save-progress 掃描的是**檔案系統**的待辦，loose-ends-scanner 掃描的是**對話內容本身**的待辦。兩者互補，一起用才完整。

建議順序：`loose-ends-scanner` → `save-progress` → 存檔完成

### 場景 3：長對話中途檢查
```
你：等等我們剛剛叉出去很多東西
Claude：🔍 掃描中... [輸出結果]
```

---

## 與 save-progress 的關係

| 工具 | 掃描對象 | 時機 |
|------|---------|------|
| **save-progress Step 1** | 檔案系統（Inbox / 專案 / 升級練等） | 對話結束 |
| **save-progress Step 4e** | 對話總結的下一步行動 | 對話結束 |
| **loose-ends-scanner** | **對話內容本身** | 隨時可用 |

loose-ends-scanner 補的是 save-progress 沒覆蓋到的盲區：**對話中提到但沒被正式記錄的散落行動**。

---

## 安全規則

- ❌ 不自動寫入，一定先展示讓你確認
- ✅ 排除已存在的項目，避免重複
- ✅ 標記來源（對話 #N loose-ends），方便追溯

## 搭配使用

- **save-progress**：存檔前先跑 loose-ends-scanner，再跑 save-progress，收尾才完整
- **flow-journal**：loose-ends-scanner 捕捉對話中的散落，flow-journal 整理長期積累的待辦
- 完整組合：`loose-ends-scanner`（掃叉出去的）＋ `save-progress`（寫日誌）＋ `flow-journal`（整理日誌）= Step 1 觀察期不漏任何一件事
