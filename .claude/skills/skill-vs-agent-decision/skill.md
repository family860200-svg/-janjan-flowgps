# Skill vs Agent 5 步決策框架

> 直接放到 `.claude/skills/skill-vs-agent-decision/` 即可使用

---

## 使用方式

對 Claude Code 說：「用 skill-vs-agent-decision 幫我評估這個任務：[任務描述]」

---

## 5 步決策流程

### Step 1：先觀察（2 週記錄，不急著建）

先用 AI 工作日誌記錄自己 2 週的所有工作任務，標記：**哪些任務一直在重複出現？**

- 重複出現 → 繼續 Step 2
- 一次性任務 → 不需要 Skill，直接做

### Step 2：重複 3 次以上 → 才考慮建 Skill

前 2 次可能是巧合，第 3 次才看得出真實模式。

當同樣的指示重複 3 次以上 → 把那段說明整理成 Skill 的工作說明書。

### Step 3：用 DRIP 矩陣判斷優先順序

| 象限 | 定義 | 行動 |
|------|------|------|
| **D（耗能 + 低產值）** | 花力氣、賺不到錢 | 優先委派或自動化 |
| **R（耗能 + 高產值）** | 花力氣、賺到錢 | Skill 化，解放自己去做 P |
| **I（充電 + 低產值）** | 很爽、賺不到錢 | 暫時保留，不急著動 |
| **P（充電 + 高產值）** | 很爽、賺到錢 | 親自做，不要外包給 Skill/Agent |

### Step 4：Skill 跑順了，才評估要不要升 Agent

**升 Agent 的三個條件**（同時滿足才升）：
1. 執行頻率超高（每週要跑好幾次）
2. 需要自主觸發（你不希望每次都下指令）
3. 規則明確（不需要你每次判斷）

**不需要升的情況**：Skill 已夠用、執行頻率低、每次都需要你的判斷。

### Step 5：需求不明確，先不動

還在「到底需不需要？」的猶豫中，答案通常是：還不到時候。

> 先觀察，讓任務告訴你答案。

---

## 快速檢核表

□ 這個任務重複出現 3 次以上了嗎？
□ 它是 D 或 R 象限（耗能）嗎？
□ 如果建 Skill，我能在 2 週內跑順它嗎？
□ 我希望 AI 自動觸發它（不需要我下指令）嗎？
□ 觸發規則是否足夠明確，不需要我每次判斷？

→ 前兩題 ✅ + 後三題都 ✅ = 升 Agent
→ 前兩題 ✅ + 後三題有 ✗ = 先做 Skill
→ 第一題 ✗ = 先不動，繼續觀察
