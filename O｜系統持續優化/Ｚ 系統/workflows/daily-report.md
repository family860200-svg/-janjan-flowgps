# 日報 Workflow

> 當 Claude 收到「日報」指令時，自動執行以下步驟。

---

## 步驟一：確認時間

```bash
date
```

---

## 步驟二：讀取 FlowGPS 現狀

1. 讀取 `F｜行動聚焦漏斗/玩家待辦任務.md`，解析所有待辦項目。
2. 讀取 `F｜行動聚焦漏斗/對話摘要/` 資料夾，找出今日或最近的摘要。

---

## 步驟三：分類整理

根據待辦任務的區塊標題，依截止日 / 優先級分類：

| 分類 | 來源 |
|------|------|
| 今日必做 | 🔴 主線（工作）＋ 有明確截止日的家庭財務 |
| 重要排程 | 🔴 主線（手工書、培力、Threads）＋ 📅 重要日期（含子任務，14天內到期） |
| 快速處理 | 🟠 系統維護 ＋ 🟡 健康日常 |
| 支線任務 | 🔵 空檔推進 |
| 已完成 | ✅ 已完成（僅統計數量） |

> **注意**：`📅 重要日期` 區塊中，凡事件日期距今 14 天以內、或子任務有明確截止日，均須納入報告。子任務（縮排項）需明確列出，不可因父項無截止日而漏報。

## 步驟五：輸出 daily-data.json

將整理後的結構化資料寫入工作目錄下的 `daily-data.json`，格式如下：

```json
{
  "generatedAt": "2026-04-03T12:00:00",
  "tasks": {
    "urgent": [
      { "title": "任務名稱", "source": "FlowGPS", "dueDate": "2026-04-10" }
    ],
    "important": [],
    "quick": [],
    "backlog": []
  },
  "schedule": [
    { "time": "09:00", "title": "行程名稱" }
  ],
  "stats": {
    "total": 20,
    "done": 8,
    "pending": 12
  }
}
```

**注意**：寫入完成後不需要告知使用者，直接讓 web app 自動偵測匯入。

---

## 步驟六：口頭報告

用簡潔的格式向指揮官報告：

1. 今天幾件待辦（依艾森豪矩陣分類）
2. 有沒有逾期或即將到期的項目
3. 建議今天先處理哪些事

---

## Workflow 自我更新規則

每次執行後，如果發現以下情況，直接修改本文件：

- 分類邏輯有誤 → 調整分類條件
- 格式不符實際使用 → 更新輸出格式
- 發現更好的整理策略 → 更新步驟

更新後簡短告知改了什麼（一句話即可）。
