---

## 清理行動清單 — 觸發流程

**Mode A（手動）** — JANJAN 說「清理行動中心」：
1. `git log --oneline -5` 確認未重複已提交
2. 讀 `F｜行動聚焦漏斗/玩家待辦任務.md`
3. 掃描 `[x]` 完成項 → 按 P0/P1/P2 分組
4. 建（或 append）`W｜八大財富積累/📔｜成果日記／反思/行動記錄/YYMMDD_完成項目歸檔.md`
5. 從待辦清單剪下 `[x]`
6. 掃過期項目 → Binary 清單問 JANJAN（延期/降級/Parking/歸檔/刪除）
7. 檢查 P0 上限 5 個、重複合併
8. 更新 game-progress.md「本月已完成」計數
9. Commit + Push
10. 簡報：歸檔 N 項、合併 M 項、待決策 K 項

**Mode B（定期 週日）** — Mode A 前 6 步 + Binary 清單直接回報阿助：
- ≤ 5 題是非題
- 24h 未回累計，3 次後自動預設執行（降級/歸 Parking）
- LOG.md 記錄「自動預設」操作

**紀律**：絕不在匯報問開放題，永遠給 Binary 或 5 選 1。寧可我多做決定，不讓 JANJAN 多做判斷。

---

## Skills 認領巡檢 SOP

1. 列出 `.claude/skills/` 所有 skill 資料夾
2. 掃描 `L｜資源放大槓桿/AI團隊/*/TOOLS.md` 提取已登記 skill
3. 比對差異 → 產出「未認領 Skills 清單」
4. 回報阿助，由阿助分配給對應員工
5. 更新 MEMORY.md 記錄巡檢時間

---

## 每週三面向價值回報（MANDATORY）

每週回報時，必須回答以下三個問題：

1. **讓 JANJAN 生活更輕鬆的建議**：有什麼可以自動化、委派、減少摩擦力的？
2. **讓 JANJAN 工作更有效率的建議**：有什麼流程可以優化、工具可以升級、重複工作可以消除的？
3. **讓 JANJAN 賺更多錢的建議**：有什麼可以提升轉換率、增加收入、開發新來源的？

每項至少一個具體、可執行的建議。不能回答「無」。
